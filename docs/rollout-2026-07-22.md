# CI harmonization rollout evidence — 2026-07-22

## Scope and final contract

The active fleet is HomeNet, Bambu, Zion, Blueprint, Lightroom, Seraph, Warren, CloudOps,
Contracts, Atlas, and Eats. Clash, ClashROI, Gabriel, Morpheus, and Unity are retired and excluded.

Each active repository has one SHA-pinned caller with a closed lowercase profile. The required
context is `repo-check / repo-check`; the repository's native gate and strict v2 receipt remain
authoritative. The organization ruleset requires a current-base pull request, resolved threads,
squash/linear history, and the shared required context with no bypass actor.

The reusable execution repository must remain public while Bambu is public. GitHub permits public
callers to use reusable workflows only from public repositories. This repository contains no
application secrets or held-out sentinel tests; protected acceptance oracles require a separate
private repository.

### Atlas four-core decision

The standard Linux job samples were 293 s, 287 s, and 273 s (287 s median). The capped four-core
runner samples were [250 s](https://github.com/njrun1804-cc/atlas/actions/runs/29962793540/job/89068707064),
[245 s](https://github.com/njrun1804-cc/atlas/actions/runs/29962793540/job/89069777712), and
[239 s](https://github.com/njrun1804-cc/atlas/actions/runs/29962793540/job/89070652277) (245 s median).
The 14.6% improvement missed the 25% adoption threshold, so the canary PR was closed without merge,
the hosted runner was deleted, and the Actions overage hard-stop budget was returned to $0.

## Phase-2 advisory security canary status

The phase-2 advisory security canary (CodeQL plus dependency-review, both non-required) stopped
at Bambu for a platform reason, not a pending decision: **Bambu is public, and both checks
require GitHub Advanced Security on private repositories.** Nine of the eleven active
repositories (Atlas, Blueprint, Contracts, Eats, HomeNet, Lightroom, Seraph, Warren, Zion) are
private; only Bambu and this repository are public.

Verified empirically on 2026-07-24 by enrolling Eats and Blueprint and observing the failures.
Both PRs were closed once the cause was established (`njrun1804-cc/eats#43`,
`njrun1804-cc/Blueprint#45`):

- `dependency-review` fails immediately: "Dependency review is not supported on this repository.
  Please ensure that Dependency graph is enabled along with GitHub Advanced Security."
- `codeql` fails during analysis: "Resource not accessible by integration."

Neither repository reports `security_and_analysis.advanced_security` enabled, and enabling it on
a private repository is a paid GitHub add-on.

**Decision: closed, not open.** Do not re-attempt fan-out to private repositories. The canary
stays on Bambu, where it runs free and green. Enrolling a private repository requires purchasing
Advanced Security first — a cost decision, not a CI decision. Dependabot (which does work on
private repositories) plus Greptile review remain the coverage for the private fleet. Re-open
only if a repository becomes public or Advanced Security is purchased.

## Acceptance evidence

- Latest `main` repository CI passed in all eleven active repositories after cutover.
- `actionlint` and pedantic `zizmor` reported no findings across all active workflow directories.
- All active repositories inherit organization ruleset `19578177` and no repository-local
  replacement ruleset.
- No active repository retains a second PR workflow that directly runs the full native gate.
- OIDC package access uses immutable organization and repository IDs and read-only package roles.
- Release, deployment, package compatibility, live diagnostics, and scheduled evidence remain
  separate from the pull-request gate.
